Introduction to the JS Snippet IDE Lite

Using the Snippet IDE

Retrieving Snippets

The snippet organizer has four categories to choose from and currently has a total of seven snippets. To navigate to any of them all you do is click one of the four buttons labeled with their respective snippet category on the menu besides the insert button.

Once the menu for the snippet category you have chosen appears you can select any of the snippets that are built in or created by you from the drop down selections at the top left of the menu. Once you have chosen a snippet and it prints out to the snippet display you can press the Get Snippet button and confirm your selection by choosing yes or no. Clicking yes will insert the snippet at the last point you have selected in either of the text consoles, if you click no it will append it at the end of the document.

The Building Blocks

The following is for reference just in case you feel like making any new UI parts or adding more categories. As this project progresses we will have youtube videos that will show you how this reference list can become a handy tool to add to the functionality of this snippet IDE.

The following is a list of id's for the UI parts of the Javascript Snippet IDE.

  1. App container: js_lab_app

    The application container is a form element that holds the text consoles and application menus. It's UI is fairly lite weight and offers only the neccassary components you need to write code and retrieve snippets.

  2. Snippet menu: jsLabAppTools

    The Snippet Menu holds the category selection buttons and the insert button.

  3. Javascript console: jsLabTextConsole

    The Javascript Console is used to create and display snippets.

  4. HTML console: htmlLabTextConsole

    The HTML Console is to create and diplay the markup part of your snippets. You can also insert style elements into it as well.

The following is a list of id's that are used for UI items that are generated by an event.

  1. Snippet prompt win: _prompt_Section

    The Snippet Prompt Window is activated when you press the Get Snippet button. It retrieves the snippet you have selected form the category menu you are currently browsing. It has two options that allow you to append the snippet at the end of the document or to insert it at a select point in the consoles. To learn more please read the Retrieveing Snippets section.

  2. Executed doc win: _document

    The Executed Document Window is the window where all your code will be executed in. It's important to note that this Snippet IDE doesn't work for code that appends elements outside of this display. (Since you won't be able to view it.) This winow appears when you use the execute button on the right side of the screen.

The following are the Create and Edit menu UI parts and there id's.

  1. Save Form: saveForm

    The save form contains the inputs to create new snippets and categories. To gain access to the save form all you have to do is press the create button, and the menu will appear. The form's required blocks are dependent on which option you choose from the category drop down list. To learn more review the Creating Snippets and Categories section.

  2. Edit Form: edit-options

    The edit form contains the inputs to edit any snippets in the organizer. It's accessed by pressing the edit button at the top of the screen. The form only requires you to choose a snippet from the drop down category other than none. Please refer to the Editing Snippets section for more info.

The following are a list of id's that are used for the HTML snippet memory. To learn more about how the application stores the snippets its best to skip to the Snippets Memory Docs section, there you'll find the template for creating categories and snippets.

  1. Reg exp memory: regExp_Snippet
  2. Set int memory: setInt_Snippet
  3. Dyn win memory: dynamicWin_Snippet
  4. SVG manip memory: menu_svgManip

The following are a list of id's that are used for the category window memory.

  1. Insert win: menu_Insert
  2. Reg exp win: menu_regExp
  3. Set int win: menu_setInt
  4. Dynamic menu win: menu_dynamicMenus
  5. SVG manip win: menu_svgManip

Creating Snippets

Remember: The application has two built in text consoles, one for Javascript and the other for HTML. The snippets are printed to these consoles when you select the snippet that you need. You can also use them to code, but the consoles don't support any code help or coloring. The snippets that the organizer comes pre-packaged with are under the Snippet Categories sub topic.

Simple steps to take.

To create a snippet is simple, the button to activate the snippet creator menu is located at the top right next to the "Edit" button.

  1. Step 1:

    Choose the category you want to save the snippet to from the drop down list, or create a new category by using the input next to the drop down list located at the top of the menu.

  2. Step 2:

    The form provides text inputs for you to create snippest, below is a list of those inputs and there associated controls

    1. Snippet Category Selection. (Non editable.)
    2. Snippet Category Title Input
    3. Snippet Title Input
    4. Snippet Description Input
    5. Snippet Parameter Inputs (Add parameter button)
    6. Snippet Rule Inputs (Add rule button)
    7. Snippet Return Input
    8. Snippet Javascript Input (Paste from js console)
    9. Snippet HTML Input (Paste from html console.)

    You can add any of the snippet's values that are currently selectable from drop down list with values. It's important to remember that none of the inputs are necassary (Except for the category input options and the snippet title input.), but if you don't complete the form the snippet that is created will still have a spot for that part except that it will be blank.

  3. Step 3:

    When you are done making your snippets all you have to do is click the save button on the bottom of the form.

  4. Optional step 4:

    When you are all done you should press control "s" to save your work permanently.
    (If you don't the snippet will only survive for the browser session.)

Snippet Categories

The four built in snippet categories have examples that range from matching numbers to manipulating SVG. Below I have listed the the snippets organized by topic.

Regular Expression Snippets

The getElement() Function

This function is useful if you don't won't to use the document.getElementsBy... methods provided by Javascript. It's parameter excepts the following format:
myElement|#;.

It's a simple (value)/(command) syntax seperated by a vertical bar followed by a semi-colon.

Other commands you can use are period (.), underscore (_), or "blank". The dot means class and the underscore means name. Leaving nothing or keeping it blank means tag name. (Remember that the pound symbol (#) means id.)

The calculate() Function

The calculate(_syntax) function is used to do summations. It's parameter uses a syntax similar to the getElement() function, but instead of only one value it has three values followed by a plus or minus sign to indicate direction. Here is an example of the format 0|10|2|+;

The first number on the left is the summation start point, the second number is the number of iterations, and the final number is the amount of increment or decrement depending on the directional symbol you use before the semi-colon.

Set Interval Snippets

The slideShow() Function

The slide show interval function is simple function that iterates through a set of images nested inside the element with the id value "slideShow". As the interval executes you can easily create custom effects by replacing the setAttribute("class") from empty to your own custom class.

The Timeline Method

The Timeline snippet provides a function named addIntervalListener(_func,_interval). The method works by updating an array that stores information regarding when to execute a function. In this way you can set up multiple events that execute functions at different times using only one interval.

The method's needed functions are the updateIntervalListener() and the intervalScan() function. It has one array and on contructor function for storing information and creating new peices of interval data.

Dynamic Window Snippets

The menuSet() Function

The menuSet() function allows you to create a menu that shares a common window. In this way you can create different sets of buttons that all have a unique style of pop up menu.

The toggle() Function

The toggle function allows you to create a button the switches the view of an element between two states. In this case we have it set to hidden and displayed.

SVG Manipulation Snippets

The spokes() Function

The spokes( _svgNodeList, _center, _radius, _initialAngle, _angleIncrement ) function places a set of svg elements on an elliptical path for you. It makes designing funner and help you move away from a square box model style.

The first parameter, _svgNodeList, is the class name of the set of svg elements you want to be placed. It is very important to remember to pass a properly formatted svg element into this function or your object will not be target correctly.

Insert Objects

The IDE also has an insert button that allows you to embed content automatically. It's located to the left of the category menu up top. There are a couple of steps that go along with using the insert menu. You can review them read them below.

  1. Step 1:

    Click the appropriate checkbox and fill out it's related text box with the path of your image or object file.

  2. Step 2:

    Click generate tag snippet than a preview snippet will appear in the textarea to the right of the path inputs. Make sure your snippets are correct and than hit insert html.

Thats all there is to that except one more thing. Remember that the image or object element will placed in the HTML text console the last place you left the cursor so be sure to place the cursor in the correct place.
